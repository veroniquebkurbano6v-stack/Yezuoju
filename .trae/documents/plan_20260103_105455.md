## 问题分析与修复方案

### 问题根源

经过深入分析代码，我发现了**两个关键缺陷**：

#### 1. 检索匹配阈值过高
在 [langchain_tools.py:443-447](file:///d:/PyCharm%202025.1/project/StoryRag%20v2.0/src/langchain_tools.py#L443-L447) 中：

```python
# 只有当 fscore >= self.file_match_threshold (默认0.7) 时才进入精确检索
if fname and fscore >= self.file_match_threshold:
    # 情况A：文件名匹配成功，只在该文件下的章节中匹配
    section_data = self.file_sections.get(fname)
```

**问题**：当 `fscore < 0.7` 时，即使找到了部分匹配的文件，也不会执行精确检索，导致 `retrieved_docs` 为空。

#### 2. 历史记录回答质量差
在 [deepseek_agent.py:165-197](file:///d:/PyCharm%202025.1/project/StoryRag%20v2.0/src/deepseek_agent.py#L165-L197) 中：

```python
# 当 has_retrieved=False 或 has_matched=False 时
if not has_retrieved or not has_matched:
    # 基于历史记录回答
    history_based_prompt = f"""基于以下对话历史记录回答用户当前问题：
    ...
```

**问题**：
- 系统错误地将 `enhanced_query`（包含系统指令的字符串）传递给 `_extract_params`，而非原始用户查询
- 基于历史记录的回答逻辑过于简单，只是简单地将历史记录拼接，没有利用语义关联

### 修复方案

#### 修复1：修复 `_extract_params` 调用逻辑
**文件**: [dialogs.py:138-147](file:///d:/PyCharm%202025.1/project/StoryRag%20v2.0/backend/app/api/dialogs.py#L138-L147)

**问题**：`enhanced_query` 包含系统指令，导致 `_extract_params` 提取到错误的参数

**修改**：
```python
# 传递原始用户查询给 _extract_params，而非 enhanced_query
result = agent.chat(query_text, chat_history=chat_history)
```

#### 修复2：降低匹配阈值
**文件**: [langchain_tools.py:443-447](file:///d:/PyCharm%202025.1/project/StoryRag%20v2.0/src/langchain_tools.py#L443-L447)

**问题**：阈值0.7过高，导致很多合理的匹配被跳过

**修改**：
```python
if fname:  # 移除 fscore >= self.file_match_threshold 限制
    # 尝试在该文件下的章节中匹配
    section_data = self.file_sections.get(fname)
```

#### 修复3：改进历史记录回答逻辑
**文件**: [deepseek_agent.py:165-197](file:///d:/PyCharm%202025.1/project/StoryRag%20v2.0/src/deepseek_agent.py#L165-L197)

**问题**：历史回答未充分利用之前的检索结果

**修改**：
- 在历史记录回答时，检查是否包含之前对话的引用来源
- 如果历史回答中有具体内容（如页码引用），应该直接使用并扩展
