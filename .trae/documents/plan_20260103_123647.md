## 改进方案：让模型主动参考历史对话

### 问题分析

当前代码逻辑：
1. 当检索成功时（`has_retrieved=True`），直接使用检索结果回答
2. 检索失败时才参考历史记录

**问题**：即使检索成功，模型也可能忽略之前对话中的重要上下文。

### 修改方案

**文件**: [deepseek_agent.py](file:///d:/PyCharm%202025.1/project/StoryRag%20v2.0/src/deepseek_agent.py#L221-L252)

**修改内容**：

1. **移除检索成功时的跳过逻辑**：不再区分"检索成功"和"检索失败"两条分支

2. **始终构建历史文本**：
```python
history_text = ""
if chat_history and len(chat_history) > 0:
    for msg in chat_history[-10:]:  # 使用最近10轮对话
        role = msg.get("role", "user")
        content = msg.get("content", "")
        history_text += f"- {role}: {content[:200]}\n" if len(content) > 200 else f"- {role}: {content}\n"
```

3. **修改 `analysis_prompt`**，在检索结果后面追加历史记录：
```python
analysis_prompt = f"""用户问题：{user_input}

检索结果：
{tool_result}

【对话历史】
{history_text if history_text else "无历史记录"}

请基于以上检索结果和对话历史回答用户的问题。如果当前检索结果不足以回答，可以结合历史对话中的相关信息进行补充。

要求：
- 如果历史记录中有相关信息，优先使用并注明"根据之前的对话"
- 采用结构化格式回答
- 在回答末尾注明主要参考来源

请直接给出结构清晰的答案。"""
```

### 预期效果

- 无论检索是否成功，模型都会考虑之前的对话上下文
- 模型可以引用之前对话中的具体信息（如页码、论据等）
- 回答更加连贯，贴合对话主题
