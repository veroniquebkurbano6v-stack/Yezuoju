## 优化方案：消除重复初始化

### 问题分析
在 `main.py` 启动时，两个组件独立初始化导致重复加载：

1. **第82-84行**: `VectorSearchSystem` 创建时初始化了：
   - `VectorDatabase`
   - `EmbeddingModel`
   - `RerankerModel` (通过 `VectorRetriever`)

2. **第93行**: `DocumentRetrievalTools` 再次初始化：
   - 另一个 `VectorDatabase` 实例
   - 另一个 `EmbeddingModel` 实例
   - 另一个 `RerankerModel` 实例 (通过内部的 `VectorRetriever`)

这导致 GPU 内存浪费、启动时间延长、以及日志重复打印。

### 优化方案

#### 方案：复用已初始化的组件

**修改文件**: `src/langchain_tools.py`

1. **新增可选参数**: 在 `DocumentRetrievalTools.__init__()` 中添加可选的组件复用参数
2. **修改 `get_langchain_tools`**: 支持传入已初始化的组件
3. **修改 `main.py`**: 传递已初始化的组件给 `DocumentRetrievalTools`

#### 实施步骤

1. **修改 `DocumentRetrievalTools.__init__`**:
   - 添加可选参数: `vector_db`, `embedding_model`, `reranker_model`
   - 如果组件已传入，直接复用；否则创建新实例

2. **修改 `get_langchain_tools`**:
   - 添加相同的可选参数，支持组件复用

3. **修改 `main.py`**:
   - 在第93行传递 `app.state.vector_system` 的组件给 `DocumentRetrievalTools`

#### 代码变更

```python
# langchain_tools.py - DocumentRetrievalTools.__init__
def __init__(self, vector_db_path: str, embedding_model_name: str = "intfloat/multilingual-e5-large",
             vector_db: VectorDatabase = None, embedding_model: EmbeddingModel = None, 
             reranker_model: RerankerModel = None):
    """初始化检索工具"""
    self.vector_db_path = vector_db_path
    # 复用已初始化的组件，或创建新实例
    self.vector_db = vector_db if vector_db is not None else VectorDatabase(vector_db_path)
    self.embedding_model = embedding_model if embedding_model is not None else EmbeddingModel(embedding_model_name)
    self.retriever = VectorRetriever(self.vector_db, self.embedding_model)
    # 复用Reranker模型
    self.reranker_model = reranker_model if reranker_model is not None else RerankerModel()
    # ... 其余代码不变

# main.py - startup_event
app.state.langchain_tools = get_langchain_tools(
    db_path, 
    embedding_model,
    vector_db=app.state.vector_system.vector_db,
    embedding_model=app.state.vector_system.embedding_model,
    reranker_model=app.state.vector_system.reranker_model
)
```

这个方案确保组件只初始化一次，同时保持代码的向后兼容性。
